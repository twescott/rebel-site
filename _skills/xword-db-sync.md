# xword-db GitHub Sync

Manages syncing the crossword word database and scripts to GitHub.

## Repo
- **URL:** https://github.com/twescott/xword-db
- **Local path:** C:\users\tiwescot\PersonalAI
- **GitHub token:** in C:\users\tiwescot\PersonalAI\config.local.json (key: github.token)
- **Username:** twescott

## Structure
- `db_split/words_NN.db` — one SQLite file per word length (generated by split_db.js)
- `db_split/affixes.db` — affixes table
- All `.js` scripts live at the repo root
- `config.local.json` is gitignored (contains credentials)
- `words.db` is gitignored (use db_split/ files instead)

## Pushing changes to GitHub

1. Export the current database into split files:
   ```
   node C:/users/tiwescot/PersonalAI/split_db.js
   ```
   This regenerates all `db_split/words_NN.db` files from `words.db`.

2. Stage, commit, and push:
   ```
   cd C:/users/tiwescot/PersonalAI
   git add -A
   git commit -m "your message"
   git push
   ```

## Pulling and restoring the database

1. Pull from GitHub:
   ```
   cd C:/users/tiwescot/PersonalAI
   git pull
   ```

2. Merge split files back into words.db:
   ```
   node C:/users/tiwescot/PersonalAI/merge_db.js
   ```

## Notes
- Never commit `words.db`, `config.local.json`, or `node_modules/` — all are gitignored.
- split_db.js is safe to re-run; it deletes and recreates all db_split files.
- merge_db.js uses INSERT OR REPLACE and is safe to run on an existing words.db.
- The split typically produces ~600 MB total across all files; no single file exceeds 100 MB so Git LFS is not needed.
